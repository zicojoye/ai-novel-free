# 启动问题修复说明

## 问题1：.env文件解析错误

### 错误信息
```
python-dotenv could not parse statement starting at line 130
```

### 原因
`.env` 文件第130-133行的CSP配置使用了错误的引号格式。`.env` 文件中，如果值包含空格，整个值需要用引号包裹。

### 修复前
```env
CSP_SCRIPT_SRC='self' 'unsafe-inline' 'unsafe-eval'
```

### 修复后
```env
CSP_SCRIPT_SRC="'self' 'unsafe-inline' 'unsafe-eval'"
```

### 说明
- 外层引号：告诉 `.env` 解析器这是一个完整的值
- 内层引号：CSP策略的实际值

---

## 问题2：python-dotenv未安装

### 错误信息
```
✗ python-dotenv (未安装)
```

### 原因
虽然 `python-dotenv` 在 `requirements.txt` 中，但安装过程可能失败。

### 解决方案
1. 运行 `修复依赖.bat` 自动安装
2. 或手动安装：
   ```cmd
   cd backend
   python -m pip install python-dotenv
   ```

---

## 问题3：脚本中文编码问题

### 错误信息
```
'��错误后重新运行' is not recognized as an internal or external command
```

### 原因
批处理脚本中的中文字符在某些PowerShell环境下可能出现编码问题。

### 修复
将错误提示中的中文替换为英文：
```batch
echo [ERROR] Configuration validation failed!
echo Please fix the errors and run again
```

## 验证修复

运行以下命令验证修复：
```cmd
cd backend
python -c "from dotenv import load_dotenv; load_dotenv(); print('OK')"
```

如果输出 `OK`，说明修复成功。
