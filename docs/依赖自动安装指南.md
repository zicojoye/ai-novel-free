# 依赖自动安装指南

## 📋 概述

AI Novel Platform 提供了完整的依赖自动安装解决方案,包括:

- ✅ 自动检测环境 (Python/Node.js/npm/pip)
- ✅ 智能跳过已安装的依赖
- ✅ 自动安装缺失的依赖
- ✅ 自动修复常见问题
- ✅ 详细的安装日志
- ✅ 安装后验证

## 🚀 快速开始

### Windows 用户

双击运行以下任意脚本即可:

```
一键安装全部依赖.bat          # 安装前后端所有依赖
```

或者单独安装:

```
backend/安装依赖.bat          # 仅安装后端依赖
frontend/安装依赖.bat         # 仅安装前端依赖
```

### Linux/macOS 用户

```bash
chmod +x 一键安装全部依赖.sh      # 添加执行权限
./一键安装全部依赖.sh              # 安装前后端所有依赖
```

或者单独安装:

```bash
cd backend
chmod +x 安装依赖.sh
./安装依赖.sh                    # 仅安装后端依赖

cd ../frontend
chmod +x 安装依赖.sh
./安装依赖.sh                   # 仅安装前端依赖
```

## 🔧 故障排除

### 后端常见问题

#### 1. Python版本过低

**错误信息**: Python版本需要3.8+

**解决方案**:
- 从 https://www.python.org/downloads/ 下载Python 3.8+
- 安装时勾选 "Add Python to PATH"

#### 2. pip不可用

**错误信息**: pip不可用

**解决方案**:
```bash
python -m ensurepip --upgrade
python -m pip install --upgrade pip
```

#### 3. 某个包安装失败

**错误信息**: ERROR: Could not find a version that satisfies the requirement

**解决方案**:
```bash
# 升级pip
pip install --upgrade pip

# 使用国内镜像源
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple <package-name>

# 或永久设置镜像源
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

#### 4. 权限错误

**错误信息**: Permission denied

**解决方案**:
```bash
# 使用 --user 参数
pip install --user <package-name>

# 或使用虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/macOS
# 或
venv\Scripts\activate     # Windows
```

### 前端常见问题

#### 1. Node.js版本过低

**错误信息**: Node.js版本需要18+

**解决方案**:
- 从 https://nodejs.org/ 下载Node.js 18 LTS版本
- 推荐使用 nvm 管理多版本Node.js

#### 2. npm安装慢或失败

**解决方案**:
```bash
# 使用国内镜像源
npm config set registry https://registry.npmmirror.com

# 或使用yarn
npm install -g yarn
yarn install
```

#### 3. node_modules损坏

**解决方案**:
```bash
# 删除node_modules和lock文件
rm -rf node_modules
rm -f package-lock.json
rm -f yarn.lock

# 重新安装
npm install
```

#### 4. 端口被占用

**错误信息**: Port 3000 is already in use

**解决方案**:
```bash
# Windows
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# Linux/macOS
lsof -ti:3000 | xargs kill -9

# 或使用其他端口
npm run dev -- --port 3001
```
