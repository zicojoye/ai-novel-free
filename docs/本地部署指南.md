# AI Novel Platform 本地部署指南

**不使用Docker,直接本地部署**

---

## 📋 前置要求检查

### 1. 环境检查

```bash
# 检查Node.js
node --version
# 需要: v20.0.0 或更高

# 检查Python
python --version
# 需要: v3.11.0 或更高

# 检查npm
npm --version
```

如果环境不符合要求,请先安装:
- Node.js: https://nodejs.org/
- Python: https://www.python.org/downloads/

---

## 🚀 部署步骤

### 步骤1: 安装前端依赖

```bash
cd ai-novel-platform/frontend

# 清理旧的依赖(如果有)
rm -rf node_modules package-lock.json

# 安装依赖
npm install
```

**预计时间**: 2-5分钟

### 步骤2: 安装后端依赖

```bash
cd ai-novel-platform/backend

# 安装Python依赖
pip install -r requirements.txt
```

**预计时间**: 3-8分钟

### 步骤3: 配置环境变量

```bash
cd ai-novel-platform

# 复制环境变量模板
cp .env.example .env

# 编辑.env文件
# Windows: notepad .env
# Linux/Mac: nano .env
```

**必须配置的变量**:
```bash
# 至少配置一个AI模型API Key
OPENAI_API_KEY=sk-xxxxxxxxxxxx
# 或者
ANTHROPIC_API_KEY=sk-ant-xxxxxxxx
# 或者
DEEPSEEK_API_KEY=sk-xxxxxxxx

# 其他使用默认值即可
```

### 步骤4: 启动服务

#### 方式A: 启动修复版(推荐,解决数据模型问题)

**启动后端**:
```bash
cd backend
python main_fixed.py
```

**启动前端**:
```bash
# 新开终端
cd frontend
npm run dev
```

#### 方式B: 启动测试版(最简单)

**启动后端**:
```bash
cd backend
python test_app.py
```

**启动前端**:
```bash
cd frontend
npm run dev
```

## 🔧 常见问题

### 问题1: 端口被占用

**症状**:
```
Error: listen EADDRINUSE: address already in use :::5173
Error: listen EADDRINUSE: address already in use :::8000
```

**解决**:

Windows:
```cmd
# 查找占用端口的进程
netstat -ano | findstr :5173
netstat -ano | findstr :8000

# 结束进程
taskkill /PID <进程ID> /F
```

Linux/Mac:
```bash
# 查找占用端口的进程
lsof -i :5173
lsof -i :8000

# 结束进程
kill -9 <进程ID>
```

### 问题2: 依赖安装失败

**前端依赖**:
```bash
cd frontend
rm -rf node_modules package-lock.json
npm install
npm install --legacy-peer-deps  # 如果有peer依赖问题
```

**后端依赖**:
```bash
cd backend
pip install --upgrade pip
pip install -r requirements.txt --force-reinstall
```

### 问题3: 模块导入错误

**症状**:
```
ModuleNotFoundError: No module named 'xxx'
```

**解决**:
```bash
cd backend
pip install -r requirements.txt
```

### 问题4: 前端空白页面

**检查**:
1. 浏览器F12打开控制台,查看错误
2. 检查网络请求是否正常
3. 检查API地址是否正确

**常见原因**:
- 后端未启动
- CORS配置问题
- API地址错误

## 📝 开发说明

### 前端开发

```bash
cd frontend

# 启动开发服务器(热重载)
npm run dev

# 构建生产版本
npm run build

# 预览构建结果
npm run preview

# 代码检查
npm run lint
```

### 后端开发

```bash
cd backend

# 启动开发服务器(热重载)
python main_fixed.py

# 或使用uvicorn直接启动
uvicorn main_fixed:app --reload --host 0.0.0.0 --port 8000
```
