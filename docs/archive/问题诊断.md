## 🔧 快速解决方案

### 方案1: 使用简化版测试 (推荐)

#### 1. 启动简化版后端
```bash
cd backend
python test_app.py
```

#### 2. 启动简化版前端

**Windows:**
```bash
cd frontend
npm run dev
```

**Linux/Mac:**
```bash
cd frontend
npm run dev
```

#### 3. 访问测试页面
打开浏览器: http://localhost:5173

---

## 📋 完整修复步骤

### 问题原因
SQLModel对JSON类型的处理在不同版本有差异,需要调整数据模型定义。

### 修复方案

1. **简化数据模型**
   - 将JSON类型改为String类型
   - 在API层进行JSON序列化/反序列化

2. **或者使用SQLite原生JSON支持**
   - 升级SQLAlchemy到最新版本
   - 使用正确的JSON类型声明

## 🚀 Docker部署 (最简单方式)

### 使用Docker可以跳过所有配置问题!

```bash
# 1. 配置环境变量
cp .env.example .env
# 编辑.env,填入API Keys

# 2. 启动服务
docker-compose up -d

# 3. 访问
# 前端: http://localhost
# 后端: http://localhost:8000
# API文档: http://localhost:8000/docs
```
