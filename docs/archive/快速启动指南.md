# 快速启动指南

## 🚀 一键启动

### Windows 用户

双击运行:

```
启动项目.bat          # 一键启动前后端所有服务
```

### Linux/macOS 用户

```bash
chmod +x 启动项目.sh
./启动项目.sh
```

## 📋 手动启动

### 1. 启动后端

**Windows:**
```bash
cd backend
启动后端.bat
```

**Linux/macOS:**
```bash
cd backend
chmod +x 启动后端.sh
./启动后端.sh
```

**手动启动:**
```bash
cd backend
python main.py
```

访问: http://localhost:8000/docs

### 2. 启动前端

**Windows:**
```bash
cd frontend
启动前端.bat
```

**Linux/macOS:**
```bash
cd frontend
chmod +x 启动前端.sh
./启动前端.sh
```

**手动启动:**
```bash
cd frontend
npm run dev
```

访问: http://localhost:3000

## 🔧 自动依赖检查

启动脚本会自动:

1. ✅ 检查依赖是否已安装
2. ✅ 已安装的依赖会跳过
3. ✅ 未安装的依赖会自动安装
4. ✅ 安装完成后自动启动服务

## 📝 访问地址

启动成功后,可以访问:

- **前端界面**: http://localhost:3000
- **后端API**: http://localhost:8000
- **API文档**: http://localhost:8000/docs
- **健康检查**: http://localhost:8000/health

## ⚙️ 配置说明

### 后端配置

编辑 `backend/app/core/config.py`:

```python
# 服务器配置
BACKEND_HOST = "0.0.0.0"  # 监听所有网卡
BACKEND_PORT = 8000       # 端口号

# 数据库配置
DATABASE_URL = "sqlite:///./ai_novel.db"

# AI服务配置
OPENAI_API_KEY = "your-api-key"  # OpenAI API密钥
ANTHROPIC_API_KEY = "your-api-key"  # Anthropic API密钥

# 向量数据库配置
QDRANT_HOST = "localhost"
QDRANT_PORT = 6333
```

### 前端配置

编辑 `frontend/src/services/api.ts`:

```typescript
export const API_BASE_URL = 'http://localhost:8000/api';
```

## 🛑 停止服务

### Windows

在对应的命令行窗口按 `Ctrl + C`

### Linux/macOS

按 `Ctrl + C` 或使用:

```bash
# 查找并杀死进程
ps aux | grep "python main.py"
kill <PID>

ps aux | grep "vite"
kill <PID>
```

## 🔍 常见问题

### 1. 端口被占用

**错误**: Address already in use

**解决方案**:

**Windows:**
```bash
netstat -ano | findstr :8000
taskkill /PID <PID> /F
```

**Linux/macOS:**
```bash
lsof -ti:8000 | xargs kill -9
```

或修改配置文件中的端口号。

### 2. 依赖安装失败

参考 [依赖自动安装指南](./依赖自动安装指南.md) 的故障排除章节。

### 3. 前端无法连接后端

检查:

1. 后端服务是否正常运行 (访问 http://localhost:8000/health)
2. 前端API配置是否正确 (`src/services/api.ts`)
3. CORS配置是否允许前端域名

### 4. 数据库错误

**错误**: Database initialization failed

**解决方案**:

```bash
# 删除现有数据库
cd backend
rm -f ai_novel.db

# 重启服务
python main.py
```
