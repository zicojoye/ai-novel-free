### 第 2 步：配置 API Keys

编辑项目根目录的 `.env` 文件，配置至少一个 AI 模型的 API Key：

```bash

# 方案 3: DeepSeek（性价比高）
```

**获取 API Keys**:
- OpenAI: https://platform.openai.com/api-keys

- Gemini: https://aistudio.google.com/app/apikey

## 🔧 常见问题

### Q1: 启动失败，提示 "ModuleNotFoundError"

**原因**: 依赖未安装

**解决方案**:
```bash
cd backend
pip install -r requirements.txt

cd ../frontend
npm install
```

---

### Q2: 前端无法连接后端

**原因**: 后端未启动或端口被占用

**解决方案**:
1. 确认后端已在运行（http://localhost:8000/health）
2. 检查 8000 端口是否被占用
3. 查看 `backend/` 目录下的错误日志

---

### Q3: AI 功能不可用

**原因**: 未配置 API Key

**解决方案**:
1. 编辑 `.env` 文件
2. 配置至少一个 API Key（见第 2 步）
3. 重启后端服务

---

### Q4: NPM 路径错误

**原因**: 系统 NPM 与内置 NPM 冲突

**解决方案**:
1. 安装系统级 Node.js: https://nodejs.org/
2. 使用系统 npm 而非内置 npm
3. 或使用 `一键初始化和安装.bat` 自动处理

## 🔒 安全建议

### 生产环境部署前

1. **修改 SECRET_KEY**:
   ```bash
   # 生成强随机密钥
   python -c "import secrets; print(secrets.token_urlsafe(32))"
   ```

2. **限制 CORS**:
   ```python
   # backend/app/core/config.py
   BACKEND_CORS_ORIGINS = [
       "https://yourdomain.com",
   ]
   ```

3. **启用 HTTPS**:
   - 使用 Nginx 反向代理
   - 配置 SSL 证书（Let's Encrypt）

4. **环境变量**:
   - 使用 `.env` 文件，不要提交到 Git
   - 生产环境使用环境变量或密钥管理服务
