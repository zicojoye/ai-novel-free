# AI Novel Platform 启动指南

## 🚀 快速启动

### Windows用户

双击运行 `start.bat` 即可自动启动所有服务!

### Linux/Mac用户

```bash
chmod +x start.sh
./start.sh
```

---

## 📋 手动启动步骤

### 1. 环境准备

#### Windows
```powershell
# 检查Node.js
node --version
# 需要 v20+

# 检查Python
python --version
# 需要 3.11+
```

#### Linux/Mac
```bash
# 检查Node.js
node --version

# 检查Python
python3 --version
```

### 2. 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑.env文件,填入API Keys
# Windows: notepad .env
# Linux/Mac: nano .env
```

**必须配置的变量:**
```bash
# AI模型API Keys (至少配置一个)
OPENAI_API_KEY=sk-xxx
ANTHROPIC_API_KEY=sk-ant-xxx
DEEPSEEK_API_KEY=sk-xxx

# 其他配置使用默认值即可
```

### 3. 安装依赖

#### 后端 (已完成 ✅)
```bash
cd backend
pip install -r requirements.txt
```

#### 前端
```bash
cd frontend
npm install
```

### 4. 启动服务

#### 方式1: 同时启动前后端

**Windows:**
```cmd
# 前端
cd frontend
start cmd /k npm run dev

# 后端
cd backend
start cmd /k python main.py
```

**Linux/Mac:**
```bash
# 前端
cd frontend
npm run dev &

# 后端
cd backend
python main.py &
```

#### 方式2: 单独启动

**启动后端:**
```bash
cd backend
python main.py
```

**启动前端 (新终端):**
```bash
cd frontend
npm run dev
```

## 🔧 常见问题

### 问题1: 端口被占用

**前端端口 5173:**
```bash
# Windows
netstat -ano | findstr :5173
taskkill /PID <PID> /F

# Linux/Mac
lsof -i :5173
kill -9 <PID>
```

**后端端口 8000:**
```bash
# Windows
netstat -ano | findstr :8000
taskkill /PID <PID> /F

# Linux/Mac
lsof -i :8000
kill -9 <PID>
```

### 问题2: 依赖安装失败

**前端依赖:**
```bash
cd frontend
rm -rf node_modules package-lock.json
npm install
```

**后端依赖:**
```bash
cd backend
pip install --upgrade pip
pip install -r requirements.txt
```

### 问题3: API调用失败

1. 检查 `.env` 文件是否配置了API Keys
2. 检查API Key是否正确
3. 检查网络连接
4. 查看后端日志输出

### 问题4: Python模块导入错误

```bash
cd backend
pip install -r requirements.txt --upgrade
```

## 🐳 Docker部署

### 前提条件
- 安装 Docker Desktop
- 确保Docker正在运行

### 启动步骤

```bash
# 1. 配置环境变量
cp .env.example .env
# 编辑.env,填入API Keys

# 2. 启动所有服务
docker-compose up -d

# 3. 查看日志
docker-compose logs -f

# 4. 停止服务
docker-compose down

# 5. 停止并删除数据
docker-compose down -v
```

### 访问地址 (Docker)
- **前端**: http://localhost
- **后端**: http://localhost:8000
- **API文档**: http://localhost:8000/docs
