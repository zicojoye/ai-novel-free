## 三、配置一致性修复

### 前后端类型统一

**修复前**:
```typescript
// 前端期望
interface Project {
  id: string
  createdAt: string
}
```

**修复后**:
```typescript
// 前端使用
interface Project {
  id: number
  created_at: string
}
```

**后端实际**:
```python
# 后端返回
class ProjectRead:
    id: int
    created_at: datetime
```

### API 路径统一

**修复前**:
```typescript
proxy: {
  '/api': {
    target: 'http://localhost:8000',
    changeOrigin: true,
  },
}
```

**修复后**:
```typescript
proxy: {
  '/api': {
    target: 'http://localhost:8000',
    changeOrigin: true,
    rewrite: (path) => path.replace(/^\/api/, '/api'),
  },
}
```

## 五、使用指南

### 1. 初始化项目

```bash
# 运行一键初始化脚本
一键初始化和安装.bat
```

### 2. 配置环境变量

编辑 `.env` 文件，配置至少一个 API Key：

```bash
# 选择一个配置
OPENAI_API_KEY=sk-your-actual-key-here
# 或
ANTHROPIC_API_KEY=sk-ant-your-key
# 或
DEEPSEEK_API_KEY=sk-your-key
```

### 3. 启动服务

**方法 1: 一键启动（推荐）**
```bash
start.bat
```

**方法 2: 手动启动**
```bash
# 终端 1: 启动后端
cd backend
python main.py

# 终端 2: 启动前端
cd frontend
npm run dev
```

### 4. 访问应用

- 前端界面: http://localhost:5173
- 后端 API: http://localhost:8000
- API 文档: http://localhost:8000/docs
