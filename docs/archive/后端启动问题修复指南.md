# 后端启动问题修复指南

## 🔍 启动前检查

启动后端前,建议先运行诊断工具:

### Windows
```bash
cd backend
诊断问题.bat
```

### Linux/macOS
```bash
cd backend
chmod +x 诊断问题.sh
./诊断问题.sh
```

诊断工具会检查:
- ✅ Python版本 (需要3.8+)
- ✅ 必需的依赖包
- ✅ 文件结构完整性
- ✅ 配置文件
- ✅ 数据库连接
- ✅ 模块导入
- ✅ 端口可用性

## 🔧 常见问题与修复

### 1. 模型导入错误

**错误信息**:
```
ImportError: cannot import name 'Project' from partially initialized module 'app.models'
```

**原因**: `app/models/__init__.py` 文件为空

**修复**: 该问题已修复,确保 `app/models/__init__.py` 包含:
```python
from .project import Project
from .chapter import Chapter
from .worldbuilding import WorldBuildingElement
from .plot import PlotPoint, PlotLine
from .knowledge import KnowledgeItem
from .agent import Agent, AgentTask
from .prompt import PromptTemplate

__all__ = [
    'Project',
    'Chapter',
    'WorldBuildingElement',
    'PlotPoint',
    'PlotLine',
    'KnowledgeItem',
    'Agent',
    'AgentTask',
    'PromptTemplate',
]
```

### 2. 数据库初始化失败

**错误信息**:
```
⚠️  Database initialization warning: [错误信息]
```

**原因**: 数据库文件权限问题或目录不存在

**修复**:
```bash
# 确保数据目录存在
cd backend
mkdir -p data
mkdir -p data/uploads
mkdir -p data/logs

# 或运行目录创建脚本
python ensure_directories.py
```

### 3. 端口被占用

**错误信息**:
```
OSError: [Errno 48] Address already in use
```

**修复方案**:

**方案1: 修改端口**
编辑 `backend/.env` 文件:
```
BACKEND_PORT=8001
```

**方案2: 释放端口**

**Windows**:
```bash
# 查找占用端口的进程
netstat -ano | findstr :8000

# 结束进程
taskkill /PID <PID> /F
```

**Linux/macOS**:
```bash
# 查找占用端口的进程
lsof -ti:8000

# 结束进程
kill -9 $(lsof -ti:8000)
```

### 4. 依赖缺失

**错误信息**:
```
ModuleNotFoundError: No module named 'fastapi'
```

**修复**:
```bash
cd backend

# 方式1: 使用自动安装脚本
python check_and_install_deps.py

# 方式2: 手动安装
pip install -r requirements.txt

# 方式3: 单独安装缺失的包
pip install fastapi uvicorn sqlmodel
```

### 5. 配置文件缺失

**错误信息**:
```
FileNotFoundError: [Errno 2] No such file or directory: '.env'
```

**修复**:

```bash
cd backend

# 复制示例配置文件
cp .env.example .env

# 编辑配置文件,添加必要的API密钥
notepad .env  # Windows
# 或
nano .env     # Linux/macOS
```

**必需配置项**:
```
# 至少配置一个AI服务的API密钥
OPENAI_API_KEY=your-openai-api-key
# 或
ANTHROPIC_API_KEY=your-anthropic-api-key
```

### 6. Python版本过低

**错误信息**:
```
❌ Python版本过低! 需要Python 3.8+
```

**修复**:
1. 下载安装Python 3.8+: https://www.python.org/downloads/
2. 或使用conda创建虚拟环境:
```bash
conda create -n ai-novel python=3.9
conda activate ai-novel
```

### 7. 模块导入失败

**错误信息**:
```
ModuleNotFoundError: No module named 'app'
```

**原因**: 从错误的目录运行

**修复**:
```bash
# 确保在backend目录下运行
cd backend

# 使用绝对路径运行
python /path/to/backend/main.py
```

### 8. CORS错误

**症状**: 前端无法连接后端

**修复**:

编辑 `backend/.env` 文件,添加前端URL:
```
BACKEND_CORS_ORIGINS=http://localhost:3000,http://localhost:5173
```

如果使用局域网访问,添加局域网IP:
```
BACKEND_CORS_ORIGINS=http://192.168.1.100:3000,http://localhost:3000
```

## 📝 开发模式 vs 生产模式

### 开发模式 (默认)
- 自动重载 (代码修改后自动重启)
- 详细日志输出
- 允许所有CORS来源

配置: `.env` 文件中设置:
```
APP_ENV=development
```

### 生产模式
- 不自动重载
- 最小日志输出
- 限制CORS来源

配置: `.env` 文件中设置:
```
APP_ENV=production
```

启动命令:
```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
```

## 🐛 调试技巧

### 1. 启用详细日志
编辑 `.env`:
```
LOG_LEVEL=DEBUG
```

### 2. 查看数据库查询
编辑 `.env`:
```
LOG_LEVEL=DEBUG
```

数据库SQL查询会打印到控制台

### 3. 测试单个API
```bash
# 健康检查
curl http://localhost:8000/health

# 获取所有项目
curl http://localhost:8000/api/projects

# 创建项目
curl -X POST http://localhost:8000/api/projects \
  -H "Content-Type: application/json" \
  -d '{"name":"测试项目","description":"测试"}'
```

### 4. 使用Python交互式调试
```bash
python
>>> from app.core.config import settings
>>> print(settings.DATABASE_URL)
>>> from app.models import Project
>>> print(Project.__name__)
```
