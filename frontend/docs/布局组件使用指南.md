# 布局组件使用指南

AI Novel Platform 布局组件提供了一套完整的响应式布局系统。

---

## 📦 组件列表

### 1. Layout - 主布局组件

整个应用的主布局容器,包含顶部导航、侧边栏和主内容区。

**使用示例:**
```tsx
import Layout from '@/components/layout/Layout'

function App() {
  return (
    <Layout>
      {/* 页面内容 */}
    </Layout>
  )
}
```

---

### 2. Sidebar - 侧边栏组件

可折叠的侧边栏导航,支持桌面端和移动端。

**Props:**
- `collapsed?: boolean` - 是否折叠
- `onCollapse?: () => void` - 折叠回调

**使用示例:**
```tsx
import { Sidebar } from '@/components/layout'

<Sidebar 
  collapsed={isCollapsed}
  onCollapse={() => setIsCollapsed(!isCollapsed)}
/>
```

**特点:**
- ✅ 折叠/展开动画
- ✅ 当前项目信息显示
- ✅ 项目锁定状态提示
- ✅ 响应式设计

---

### 3. Header - 顶部导航栏

包含 Logo、项目选择器和用户菜单。

**Props:**
- `onCreateProject?: () => void` - 创建项目回调

**使用示例:**
```tsx
import { Header } from '@/components/layout'

<Header onCreateProject={handleCreateProject} />
```

**特点:**
- ✅ 项目快速切换
- ✅ 用户菜单下拉
- ✅ 新建项目按钮
- ✅ 响应式设计

---

### 4. MobileNav - 移动端导航

移动端专用的抽屉式导航菜单。

**Props:**
- `isOpen: boolean` - 是否打开
- `onClose: () => void` - 关闭回调

**使用示例:**
```tsx
import { MobileNav } from '@/components/layout'

<MobileNav 
  isOpen={mobileNavOpen}
  onClose={() => setMobileNavOpen(false)}
/>
```

---

### 5. MainContent - 主内容区

页面主内容容器,限制最大宽度。

**Props:**
- `children: React.ReactNode` - 子元素
- `className?: string` - 额外样式类

**使用示例:**
```tsx
import { MainContent } from '@/components/layout'

<MainContent>
  {/* 页面内容 */}
</MainContent>
```

---

### 6. Container - 容器组件

响应式容器,可设置不同宽度。

**Props:**
- `children: React.ReactNode` - 子元素
- `size?: 'sm' | 'md' | 'lg' | 'xl' | 'full'` - 容器大小 (默认: lg)
- `className?: string` - 额外样式类

**使用示例:**
```tsx
import { Container } from '@/components/layout'

<Container size="md">
  <p>中等宽度的内容</p>
</Container>
```

**尺寸对应:**
- `sm`: max-w-2xl
- `md`: max-w-4xl
- `lg`: max-w-6xl
- `xl`: max-w-7xl
- `full`: max-w-full

---

### 7. PageHeader - 页面头部组件

统一的页面头部,包含标题、副标题、面包屑和操作按钮。

**Props:**
- `title: string` - 页面标题
- `subtitle?: string` - 副标题
- `actions?: React.ReactNode` - 操作按钮
- `breadcrumbs?: Array<{ label: string; href?: string }>` - 面包屑
- `className?: string` - 额外样式类

**使用示例:**
```tsx
import { PageHeader } from '@/components/layout'

<PageHeader
  title="项目设置"
  subtitle="管理您的项目配置"
  breadcrumbs={[
    { label: '首页', href: '/' },
    { label: '项目' },
    { label: '设置' },
  ]}
  actions={
    <button className="px-4 py-2 bg-primary rounded-lg">
      保存
    </button>
  }
>
```

**特点:**
- ✅ 面包屑导航
- ✅ 响应式布局
- ✅ 操作按钮区域
- ✅ 可定制样式

---

### 8. Card - 卡片组件

通用的卡片容器,支持头部和底部。

**Props:**
- `children: React.ReactNode` - 卡片内容
- `title?: string` - 卡片标题
- `headerActions?: React.ReactNode` - 头部操作按钮
- `footer?: React.ReactNode` - 底部内容
- `className?: string` - 额外样式类

**使用示例:**
```tsx
import { Card } from '@/components/layout'

<Card
  title="统计信息"
  headerActions={<button>刷新</button>}
  footer={<div className="text-sm text-muted-foreground">最后更新: 2024-01-01</div>}
>
  <p>卡片内容</p>
</Card>
```

---

### 9. Grid - 网格布局组件

响应式网格布局,自动适应不同屏幕尺寸。

**Props:**
- `children: React.ReactNode` - 子元素
- `cols?: 1 | 2 | 3 | 4 | 6 | 12` - 列数 (默认: 3)
- `gap?: 1 | 2 | 3 | 4 | 6 | 8` - 间距 (默认: 4)
- `className?: string` - 额外样式类

**使用示例:**
```tsx
import { Grid, Card } from '@/components/layout'

<Grid cols={3} gap={4}>
  <Card>项目 1</Card>
  <Card>项目 2</Card>
  <Card>项目 3</Card>
</Grid>
```

**列数对应:**
- `1`: 1列 (所有屏幕)
- `2`: 1列(移动) → 2列(桌面)
- `3`: 1列(移动) → 2列(平板) → 3列(桌面)
- `4`: 1列(移动) → 2列(平板) → 3列(大屏) → 4列(超大屏)
- `6`: 2列(移动) → 3列(平板) → 4列(大屏) → 6列(超大屏)
- `12`: 1列 → 2列 → 3列 → 4列 → 6列 → 12列

---

### 10. TabNavigation - 标签页导航组件

支持多种样式的标签页导航。

**Props:**
- `tabs: Tab[]` - 标签页数组
  - `id: string` - 标签ID
  - `label: string` - 标签文本
  - `icon?: string` - 标签图标
  - `disabled?: boolean` - 是否禁用
  - `content?: React.ReactNode` - 标签内容
- `defaultTab?: string` - 默认标签
- `onChange?: (tabId: string) => void` - 切换回调
- `variant?: 'default' | 'pills' | 'underline'` - 样式变体 (默认: default)
- `className?: string` - 额外样式类

**使用示例:**
```tsx
import { TabNavigation } from '@/components/layout'

<TabNavigation
  defaultTab="overview"
  onChange={(tabId) => console.log('切换到:', tabId)}
  variant="pills"
  tabs={[
    {
      id: 'overview',
      label: '概览',
      icon: '📊',
      content: <div>概览内容</div>
    },
    {
      id: 'settings',
      label: '设置',
      icon: '⚙️',
      content: <div>设置内容</div>
    },
    {
      id: 'help',
      label: '帮助',
      icon: '❓',
      disabled: true,
    },
  ]}
/>
```

**样式变体:**
- `default`: 下划线样式 (默认)
- `pills`: 胶囊样式
- `underline`: 下划线样式 (同 default)

## 📐 设计规范

### 响应式断点
- 移动端: < 768px (md)
- 平板: 768px - 1024px (lg)
- 桌面: 1024px - 1280px (xl)
- 大屏: > 1280px

### 间距系统
- gap-1: 0.25rem (4px)
- gap-2: 0.5rem (8px)
- gap-3: 0.75rem (12px)
- gap-4: 1rem (16px) - **默认**
- gap-6: 1.5rem (24px)
- gap-8: 2rem (32px)

## 📝 注意事项

1. **Layout 组件必须在最外层**: 所有页面都应该包裹在 Layout 组件中
2. **响应式设计优先**: 所有布局组件都支持响应式,优先使用 Grid 而非固定宽度
3. **移动端体验**: 侧边栏在移动端会自动隐藏,使用 MobileNav 替代
4. **语义化 HTML**: 使用正确的语义化标签,提高可访问性

---

## 🎯 最佳实践

1. **统一使用布局组件**: 避免手动编写重复的布局代码
2. **保持一致性**: 使用相同的间距、容器大小保持页面风格一致
3. **响应式优先**: 确保所有页面在移动端都能正常使用
4. **组件复用**: 通过 props 定制布局组件,而非创建新组件
