# 前端代理配置说明

## 📋 概述

AI Novel Platform 前端使用 Vite 的代理功能,在开发环境中将 API 请求转发到后端服务,避免跨域问题。

## 🔧 配置文件

### 1. 环境变量文件

#### `.env` (默认配置)
```env
# API配置
VITE_API_BASE_URL=/api
VITE_API_TIMEOUT=60000

# 前端服务器端口
VITE_PORT=3000
```

#### `.env.development` (开发环境)
```env
# 使用代理
VITE_API_BASE_URL=/api
VITE_API_TIMEOUT=60000
VITE_PORT=3000
VITE_ENABLE_DEBUG=true
```

#### `.env.production` (生产环境)
```env
# 生产环境直接连接后端
VITE_API_BASE_URL=http://your-backend-domain.com/api
VITE_API_TIMEOUT=30000
VITE_PORT=80
VITE_ENABLE_DEBUG=false
```

### 2. Vite 配置 (`vite.config.ts`)

```typescript
server: {
  port: 3000,
  host: true, // 支持局域网访问
  proxy: {
    '/api': {
      target: 'http://localhost:8000',  // 后端地址
      changeOrigin: true,               // 修改请求头中的origin
    },
  },
}
```

## 🔄 代理工作原理

### 开发环境

```
前端 (localhost:3000)
    ↓ API请求 /api/projects
Vite 代理
    ↓ 转发到
后端 (localhost:8000/api/projects)
```

### 生产环境

```
前端 (nginx/apache)
    ↓ API请求 /api/projects
直接请求
    ↓
后端 (http://backend.com/api/projects)
```

## 🌐 局域网访问配置

### 1. 后端配置

确保后端监听所有网卡:

编辑 `backend/.env`:
```env
BACKEND_HOST=0.0.0.0
BACKEND_PORT=8000
```

### 2. 前端配置

Vite 配置已启用局域网访问:

```typescript
server: {
  host: true, // 监听所有地址
}
```

### 3. 访问地址

查看本机IP地址:

**Windows:**
```bash
ipconfig
```

**Linux/macOS:**
```bash
ifconfig
# 或
ip addr show
```

然后在其他设备上访问:

- 前端: `http://192.168.1.100:3000`
- 后端: `http://192.168.1.100:8000`

### 4. CORS配置

如果遇到跨域问题,配置后端CORS:

编辑 `backend/.env`:
```env
BACKEND_CORS_ORIGINS=http://localhost:3000,http://192.168.1.100:3000
```

或修改 `backend/app/core/config.py`:
```python
BACKEND_CORS_ORIGINS: List[str] = [
    "http://localhost:3000",
    "http://192.168.1.100:3000",
    "http://localhost:5173",
]
```

## 🔍 代理规则

### API代理

所有 `/api` 开头的请求都会被代理到后端:

```typescript
// 前端请求
api.get('/api/projects')

// 实际请求到后端
http://localhost:8000/api/projects
```

### 其他代理

```typescript
proxy: {
  '/api': {
    target: 'http://localhost:8000',
    changeOrigin: true,
  },
  '/health': {
    target: 'http://localhost:8000',
    changeOrigin: true,
  },
  '/docs': {
    target: 'http://localhost:8000',
    changeOrigin: true,
  },
  '/static': {
    target: 'http://localhost:8000',
    changeOrigin: true,
  },
}
```

## 🛠️ 常见配置场景

### 场景1: 前后端同机开发

配置:
```
前端: localhost:3000
后端: localhost:8000
```

前端 `.env`:
```env
VITE_API_BASE_URL=/api
```

### 场景2: 后端在远程服务器

配置:
```
前端: localhost:3000
后端: http://192.168.1.200:8000
```

前端 `.env`:
```env
VITE_API_BASE_URL=/api
```

`vite.config.ts`:
```typescript
proxy: {
  '/api': {
    target: 'http://192.168.1.200:8000',
    changeOrigin: true,
  },
}
```

### 场景3: 前端开发,后端已部署

配置:
```
前端: localhost:3000
后端: https://api.example.com
```

前端 `.env`:
```env
VITE_API_BASE_URL=https://api.example.com/api
```

此时不需要代理配置。

### 场景4: 生产环境部署

配置:
```
前端: https://example.com
后端: https://api.example.com
```

前端 `.env.production`:
```env
VITE_API_BASE_URL=/api
```

使用 nginx 反向代理:
```nginx
server {
    listen 80;
    server_name example.com;

    # 前端静态文件
    location / {
        root /var/www/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## ⚠️ 常见问题

### 1. 代理不生效

**原因**: 环境变量未正确加载

**解决**:
```bash
# 重启开发服务器
npm run dev
```

### 2. CORS错误

**错误信息**:
```
Access to XMLHttpRequest at 'http://localhost:8000' from origin 'http://localhost:3000' has been blocked by CORS policy
```

**解决**:
1. 确保使用了代理 (`VITE_API_BASE_URL=/api`)
2. 配置后端CORS允许前端域名
3. 检查 `changeOrigin: true` 是否设置

### 3. WebSocket连接失败

**错误信息**:
```
WebSocket connection to 'ws://localhost:8000' failed
```

**解决**:
确保 `vite.config.ts` 中配置了:
```typescript
proxy: {
  '/api': {
    ws: true,  // 启用WebSocket代理
  },
}
```

### 4. 局域网无法访问

**原因**: 防火墙阻止

**解决**:

**Windows**:
```bash
# 允许端口通过防火墙
netsh advfirewall firewall add rule name="Vite Dev Server" dir=in action=allow protocol=TCP localport=3000
```

**Linux/macOS**:
```bash
# 检查防火墙
sudo ufw status
sudo ufw allow 3000
```

### 5. 请求超时

**原因**: 后端响应慢或网络问题

**解决**:

增加超时时间:
```env
VITE_API_TIMEOUT=120000
```

或 `vite.config.ts`:
```typescript
proxy: {
  '/api': {
    proxyTimeout: 120000,
  },
}
```

## 📝 API客户端配置

### 基础配置 (`src/lib/api.ts`)

```typescript
const api = axios.create({
  baseURL: API_BASE_URL,  // 从环境变量读取
  timeout: API_TIMEOUT,
  headers: {
    'Content-Type': 'application/json',
  },
})
```

### 请求拦截器

自动添加:
- Token认证
- 项目ID
- 时间戳防缓存
- 自定义请求头

### 响应拦截器

统一处理:
- 401未授权 (自动跳转登录)
- 403禁止访问
- 404资源不存在
- 422参数验证错误
- 429请求过多
- 500服务器错误

- 代理只在开发环境生效
